结构

小程序框架 的目标是尽可能以简单、高效的方式让开发者在微信中有原生App体验的服务。

框架

整个小程序框架系统分为：

  1. 逻辑层(App Service)
  2. 视图层(View)

小程序在视图层与逻辑层间提供了数据传输和事件系统，核心是一个响应的数据绑定系统。

当做数据修改的时候，只需要在逻辑层修改数据，视图层就会做相应的更新。（类似 Vue.js | React.js）

对比

传统Web 微信小程序
结构 HTML WXML
样式 CSS WXSS
逻辑 JavaScript JavaScript
配置 无 JSON

通过以上对比得出，传统Web 是3层结构，而 微信小程序 是4层结构，多了一层 配置.json

样式

暗黑模式

  1. 开启暗黑模式

    • 在 app.json 中配置 "darkmode": true

    •   // app.json
  {
      ...
      "darkmode": true
  }

  2. 配置主题文件

    • 新建主题配置文件 theme.json，并在 app.json 中配置路径引入

    •   // app.json
  	{
  	    ...
  	    "themeLocation": "theme.json"
  	}
      • light 正常模式
      • dark 暗黑模式
    •   // theme.json
  "light": {
  	"navBackgroundColor": "#ffffff",
  	"navTextStyle": "black"
  },
  "dark": {		
  	"navBackgroundColor": "#000000",
    "navTextStyle": "white"
  }
  /* 必须存在 light 和 dark 两个属性，里层命名自定义，没有严格要求 */

  3. 在 app.json 中应用配置属性

    • 在配置属性以 @ 开头拼接 theme.json 中自定义的名字写入配置：
//app.json

{
  ...
  "window":{
    "navigationBarTitleText": "Weixin",
    "navigationBarTextStyle":"@navTextStyle",
    "navigationBarBackgroundColor": "@navBackgroundColor"
  },
  ...
  "darkmode": true,
  "themeLocation": "theme/theme.json"
}

  1. wxss样式适配暗黑模式

    • wxss 中，支持通过媒体查询 prefers-color-scheme 适配不同主题。

      • 使用CSS选择器@media (prefers-color-scheme: dark)

    • 如下样式会在正常模式下页面背景为灰白色，暗黑模式下为黑色。

      •   /* 正常模式下应用的样式 */
  page{
      background: #f1f1f1;
  }

  /* 暗黑模式下应用的样式 */
  @media (prefers-color-scheme: dark) {
    page{
      background: #000000;
    }
  }

<!-- 2. <view>相当于<div>标签 -->

  • 可以在双大括号 {{ }} 中加入表达式 ->“语句”

列表渲染

wx:for

  1. ”{{数组或者对象}}"：wx:for
  2. “循环项的名称”：wx:for-item
  3. “循环项的索引”：wx:for-index

可以将 wx:for 用在<block/>标签上，以渲染一个包含多节点的结构块。

Note: wx:for-item & wx:for-index的默认值为item(当前元素)和index(该块在渲染列表中的顺序)

⚠️ Caution 当出现嵌套循环绑定的时，尤其要注意绑定的名称（wx:for-item & wx:for-index）不要重复。

<!--wxml-->
<view wx:for="{{array}}">
    {{item}}
</view>

<view wx:for="{{objectArray}}"
      wx:for-item="item"
      wx:for-index="index">
    索引{{index}}
    ---
    值{{item.name}}
</view>
// page.js
Page({
  data: {
    numberArray: [1, 2, 3, 4, 5],
		objectArray: [
			{id: 0, name: "name_0"},
			{id: 1, name: "name_1"},
			{id: 2, name: "name_2"}
		]
})

wx:key

指定循环中具体项目。

”{{唯一的值}}"：wx:key

wx:key 的值以两种形式提供：

  1. 字符串，代表在 for 循环的 array 中 item 的某个 property，该 property 的值唯一且不能动态改变。
    • {{objectArray}} ⟹ wx:key="unique"
  2. *this 代表在 for 循环中的item本身，item唯一。
    • {{numberArray}} ⟹ wx:key="*this"
    • 表示数组是一个普通数组：*this是一个循环项

当数据改变触发渲染层重新渲染的时候，会校正带有key的组件，框架会确保他们被重新排序，而不是重新创建，以确保使组件保持自身的状态，并且提高列表渲染时的效率。

Note: 可以使用wx:key="*this"来去除wx:for中的 Warning 。

⚠️ Caution wx:key指定的标签不存在时，系统也不会报错。

<view wx:for="{{objectArray}}" wx:key="unique"> {{item.id}}</view>

<view wx:for="{{numberArray}}" wx:key="*this"> {{item}} </view>
// page.js
Page({
	data: {
		objectArray: [
			{id: 2, unique: 'unique_2'},
			{id: 1, unique: 'unique_1'},
			{id: 0, unique: 'unique_0'},
		],
		numberArray: [1, 2, 3, 4]
	},
});

条件渲染

wx:if

判断是否需要渲染该代码块：

<!--wxml-->
<view wx:if="{{view == 'WEBVIEW'}}"> WEBVIEW </view>
<view wx:elif="{{view == 'APP'}}"> APP </view>
<view wx:else="{{view == 'MINA'}}"> MINA </view>
// page.js
Page({
  data: {
    view: 'MINA'
  }
})

📝 Note wx:if vs hidden

  • 因为 wx:if 之中的模板也可能包含数据绑定，所以当 wx:if 的条件值切换时，框架有一个局部渲染的过程，因为它会确保条件块在切换时销毁或重新渲染

  • 同时 wx:if 也是惰性的，如果在初始渲染条件为 false，框架什么也不做，在条件第一次变成真的时候才开始局部渲染

  • hidden 就简单的多，组件始终会被渲染，只是简单的控制显示与隐藏

    • hidden
    • hidden = "{{condition}}"

<block>

将多个组件包装起来，并在上边使用 wx:if 控制属性。

Note: <block/> 并不是一个组件，它仅仅是一个包装元素，不会在页面中做任何渲染，只接受控制属性。

<block wx:if="{{view == 'MINA'}}">
	<view>view1</view>
  <view>view2</view>
</block>

WXSS

WXSS（WeiXin Style Sheets）是一套样式语言，用于描述 WXML 的组件样式，决定 WXML 的组件应该怎么显示。

  1. 小程序中不需要主动引入样式文件
  2. 需要把页面中某些元素的单位由 px 改为 rpx / %
  3. 换算关系：750rpx = 100%

尺寸单位

rpx（responsive pixel）可以根据屏幕宽度进行自适应。规定屏幕宽为750rpx。

  • 如在 iPhone6 上，屏幕宽度为375px，共有750个物理像素，则750rpx = 375px = 750物理像素，1rpx = 0.5px = 1物理像素。

CSS 原生支持calc属性

  •   view{
  	width: calc(750rpx * 100 / 375);
  }

Note: 建议开发时以iPhone6宽度为标准。

样式导入

使用@import语句可以导入外联样式表，@import后跟需要导入的外联样式表的相对路径，用;表示语句结束。

  •   @import "../../styles/common.wxss"

LESS

原生小程序不支持less，其他基于小程序的框架大体都支持，如wepy mpvue taro等。可以用以下方式来实现：

: LESS（Linear Style Sheet）是一个CSS*预处理语言

  1. 编辑器是 VS Code

  2. 安装插件 easy less

    • Auto-compile LESS to CSS on save

  3. 在 VS Code 的设置中配置：

    •   "less.compile": {
  	"outExt": ".wxss"
  }

  4. 在要编写样式的地方，新建less文件，如index.less，然后正常编辑即可

    •   /* 1 定义less变量 */
  @color:yellow;
  text{
    /* 2 使用变量 */
    color: @color;
  }

  .view{
    .vie1{
      text{
        color: red;
      }
    }
  }

Note: 从开发者工具 1.05.2109101 以上开始，支持以编译插件的形式，扩展编译功能。

使用：

  1. 旧项目：

    • 在 project.config.json 文件中，修改 setting 下的useCompilerPlugins字段为["typescript"]，即可开启工具内置的 typescript 编译插件
    • 如需同时开启less 编译插件，可将该字段修改为 ["typescript", "less"]

  2. 新建项目：

    • 可在创建小程序项目时，选择对应的语言模板。 目前支持的语言模板有：
    • TypeScript TypeScript + Less TypeScript + Sass

  2. 通过事件源对象e.detail.value获取输入字符串
  3. 把输入框的值赋值到data中
    • *****: 不能直接使用this.data.num = ~
布局
<!--wxml-->
<input type="text" bind:input="handleInput"/>
<view>
    {{inputStr}}
</view>
// page.js
Page({
  data:{
    inputStr:""
  }
  handleInput(e) {
   this.setData({
	  inputStr:e.detail.value;
   })
  }
})

媒体组件

image

图片。支持 JPG、PNG、SVG、WEBP、GIF 等格式。支持懒加载。

默认尺寸 320px × 240px 。

⚠️ Caution 微信小程序使用的是 image 标签，这和HTML使用的 img 不同

  1. lazy-load：图片懒加载，在即将进入一定范围（上下三屏）时才开始加载
  2. show-menu-by-longpress：长按图片显示菜单
  3. webp：webP支持
  4. bindload：图片载入完毕时触发
    • event.detail = {height, width}
  5. mode：图片裁剪、缩放的模式
    • scaleToFill - default
    • aspectFit - 保持宽高比，保证长边显示 页面轮播图常用
    • aspectFill - 保持宽高比，显示短边，截取长边 少用
    • widthFix - 保持宽高比，根据宽度缩放 常用
    • heightFix
    • etc…

swiper

滑块视图容器。其中只可放置 swiper-item 组件。

默认尺寸 100% × 150px ，swiper-item 组件中 image 默认尺寸 320px × 240px ，且尺寸无法实现由内容撑开。

  • 解决方法：寻找原图的高度和宽度，等比例为 swiper 确定高度与宽度
属性
  1. auto-play：自动连播
  2. interval：修改轮播时间 单位ms
  3. duration：动画持续时间
  4. indicator-dots：显示轮播指示点
  5. circular： 循环滚动

Navigator

页面链接。类似 <a href=""></a>。

属性

  1. url：当前小程序内的跳转链接（page格式）
  2. open-type：跳转方式
    • navigate - 保留当前页面，跳转到应用内的某个页面。但是不允许跳转到tabbar页面
    • navigateBack - 返回上一页面（类似返回值）
    • redirect - 关闭当前页面，跳转到应用内的某个页面。但是不允许跳转到tabbar页面
    • switchTab - 跳转到tabBar页面，并关闭所有非tabBar页面
    • relaunch - 关闭所有页面，打开到应用内的某个页面

⚠️ Caution 不支持相对路径，请使用绝对路径（含/）

  •   <navigator url="/url/to/page">nav</navigator>

⚠️ Caution 跳转至绑定了 tabBar 的页面时需设置open-type="switchTab"

  • 跳转到tabBar页面，并关闭其他所有非tabBar页面

模块

类似 vue 或者 react 中的自定义组件，允许我们使用自定义组件的方式来构建页面。

淘宝首页： 导航模块 展示模块 …

  • 对某些代码进行复用

创建

可在微信开发者工具中快速创建组件的文件结构，放于专用的components目录。

声明

首先要在组件的json文件中进行自定义组件声明：

// component.json
{
  "component": true
}

在原页面中：

<!-- page.wxml -->
<tabs></tabs>
// page.json
{
  "usingComponents": {
    "tabs": "/components/tabs/tabs"
    //绝对路径或相对路径
  }
}

即可使用。

编辑

这里以 自定义标签栏 （tabs）为例。

  1. 绑定点击事件 需要在 methods 中绑定
  2. 获取被点击的索引
  3. 获取原数组
  4. 对数组循环
    • 为每一个循环项选中属性改为false
    • 为当前索引添加激活效果

索引位于：e.currentTarget.dataset ，注意通过 data-index 进行传递

Warning:

  1. 页面.js 文件中，存放事件回调函数须放在 data 同层级
  2. 组件.js 文件中，存放事件回调函数须放在 methods 中

Note: Js 对复杂类型进行解构时只是复制了一份变量的引用

对于最严谨的做法，是重新拷贝一份数组，再对数组的备份进行处理，不要直接修改 this.data.数据 。

  1.  let {tabs} = this.data;
  2.  let tabs = this.data.tabs

这两者达成的效果是一致的。

<text>components/tabs/tabs.wxml</text>
<view class="tabs">
    <view class="tabs_title">
        <block wx:for="{{tabs}}" wx:key="id">
            <view
                    class="title_item {{item.isActive?'active':''}}"
                    bind:tap="handleItemTap"
                    data-index="{{index}}"
            >
    <!--这里注意添加data-index属性以传递参数-->
                {{item.name}}
            </view>
        </block>
    </view>
    <view class="tabs_content">
        <text>内容</text>
    </view>
</view>
// components.wxss
.tabs_title {
    display: flex;
}
.title_item {
    flex: 1;
    display: flex;
    justify-content: center;
    align-items: center;
}
view.active{
    color: rgb(0, 32, 29);
    border-bottom: 5rpx solid currentColor;
}
// component.js
Component({
	properties: {
		//接受父向子传递的数据
	},
	data: {
		tabs: [
			{id: 0, name: "首页", isActive: true},
			{id: 1, name: "原创", isActive: false},
			{id: 2, name: "分类", isActive: false},
			{id: 3, name: "关于", isActive: false}
		]
	},
	methods: {
		//存放事件回调函数
		handleItemTap(e) {
			// 获取索引
			let {index} = e.currentTarget.dataset;
			// 获取data中的数组
			let {tabs} = this.data;
      //等效与 let tabs = this.data.tabs;
      
      /*Note:为了不修改原来的数据，最好的办法是进行深拷贝：
      let tabs = 
      JSON.parse(JSON.stringfy(this.data.tabs));
      */
      
			console.log(tabs);
			//  对象数组的循环数组
			/* [].forEach 遍历数组，遍历的时候若修改了v,i也会导致原数组被修改*/
			tabs.forEach((v, i) => {
				v.isActive = i === index;
			})
			this.setData({
				tabs
			})
		}
	},
});

传参

父组件(页面) 通过 标签属性 的方式传递向 子组件 传递数据。

子组件 在 properties 属性中配置接收

  1. 父组件(页面)

    •   <!--传递常数-->
  <tabs aaa="123"></tabs>
  <!--传递变量-->
  <tabs tabs="{{tabs}}"></tabs>

  2. 子组件

    •   <!--component.wxml-->
  <view>{{aaa}}</view>
    •   // component.js
  Component({
  	properties: {
  		//接受父向子传递的数据的名称
  		aaa:{
  			//type 要接受的数据类型
  			type:String,
  			//value 默认值
  			value:""
  		},
      tab{
        type:Array,
        value:[]
      }
  	}
  });

优化

在 子组件 中进行的这个函数，修改的是 子组件 自己的变量

this.setData({
	tabs
})

优化目标：触发 父组件 中的自定义事件，同时传递数据。

  • 子组件 通过事件向 父组件 传递数据
    • 传递的参数 index 会被包裹成为 e.detail.index
  • 在 子组件 的事件上加入一个自定义事件

Note: 这些方法实际开发时作为一整个模块，在组件内部修改，使用时直接调用即可。

子组件：

<!--component.wxml-->
<view bind:tap="handler_child"></view>
<!--子组件中绑定具体参数-->
// component.js handler_child()
/* Component -> methods -> handler_child(e)*/
this.triggerEvent("父组件自定义事件的名称", 要传递的参数);

父组件：

<!--page.wxml-->
<component tabs="{{tabs}}" bind:itemChange="handler_parent"></component>
<!--第一个是向子组件传递的参数 第二个是父组件中相应的处理程序-->
// page.js handler_parent()
/* Page -> handler_parent(e)*/
handler_parent(e){
    //获取传递的参数
		let {index} = e.detail;
		/* the same as
		* let index = e.detail.index; */
		...
}

<slot>

在组件的WXML中可以包含<slot>节点，用于承载组件使用者提供的WXML结构。

Note: slot 标签是一个占位符插槽，等到父组件调用子组件的时候再传递标签过来，最终这些被传递的标签就会替换slot插槽的位置。

<!-- page.wxml -->
<view>
  <component-tag-name prop-a="{{dataFieldA}}" prop-b="{{dataFieldB}}">
    <view>这里是插入到组件slot中的内容</view>
  </component-tag-name>
</view>

默认情况下，一个组件的 wxml 中只能有一个 slot 。需要使用多 slot 时，可以在组件 js 中声明启用。

//component.js
Component({
  options: {
    multipleSlots: true // 在组件定义时的选项中启用多slot支持
  },
  properties: { /* ... */ },
  methods: { /* ... */ }
})

此时，可以在这个组件的wxml中使用多个 slot ，以不同的name来区分。

<!-- component.wxml -->
<view class="wrapper">
  <slot name="before"></slot>
  <view>这里是组件的内部细节</view>
  <slot name="after"></slot>
</view>

使用时，用 slot 属性来将节点插入到不同的 slot 上。

<!-- page.wxml -->
<!-- 引用组件的页面模板 -->
<view>
  <component-tag-name>
    <view slot="before">这里是插入到组件slot name=""before"中的内容</view>
    <view slot="after">这里是插入到组件slot name=""after"中的内容</view>
  </component-tag-name>
</view>

应用： Tab页面

<!-- component.wxml -->
<view class="tabs_content">
  <slot></slot>
</view>
<!-- page.wxml -->
<tabs tabs="{{tabs}}" bind:itemChange="handleItemChange">
  <!--单个slot的时候，不需要view name=""特别指定"-->
  <block wx:for="{{tabs}}" wx:key="name">
    <block wx:if="{{tabs[index].isActive}}">
      <view>{{ tabs[index].name }}</view>
    </block>
  </block>
</tabs>

定义段

定义段 类型 必 描述
properties Object Map 否 组件的对外属性，是属性名到属性设置的映射表，参见下文
data Object 否 组件的内部数据，和properties一同用于组件的模板演染
observers Object 否 组件数据字段监听器，用于监听properties和data的变化，参见数据监听器
**注意：**只存在于组件当中
methods Object 否 组件的方法，包括事件响应函数和任意的自定义方法，关于事件响应函数的使用，参见组件事件
created Fuction 否 组件生命周期函数，在组件实例刚刚被创建时执行，注意此时不能调用setData，参见组件生命周期
attached Fuction 否 组件生命周期函数，在组件实例进入页面节点树时执行，参见组件生命周期
ready Fuction 否 组件生命周期函数，在组件布局完成后执行，参见组件生命周期
moved Fuction 否 组件生命周期函数，在组件实例被移动到节点树另一个位置时执行，参见组件生命周期
detached Fuction 否 组件生命周期函数，在组件实例被从页面节点树移除时执行，参见组件生命周期

周期

分为应用生命周期和页面生命周期

应用

// app.js
App({
  onlaunch() {
    // 1. 在应用第一次启动
    /* 此时可申请获取用户个人信息 */
  },
  onShow() {
    // 2. 应用被用户看到
    /* 对应用的数据或者页面效果重置 */
  },
  onHide() {
    // 3. 应用被隐藏
    /* 暂停或清除定时器 */
  },
  onError(err) {
    // 4. 应用代码发生错误
    /* 收集用户错误信息，通过异步请求发送用户错误信息 */
  },
  onPageNotFound() {
    // 5. 页面找不到就会触发
    /* 应用第一次启动时，找不到入口页面，
    ** 无法被 wx.navigateTo 触发 */
    wx.navigateTo({
      url: "pages/404/404"
    });
  }
});
属性 类型 必填 说明
onLaunch Function 否 监听小程序初始化
onShow Function 否 监听小程序启动或切前台
onHide Function 否 监听小程序切后台
onError Function 否 错误监听函数
onPageNotFound Function 否 入口页面不存在

页面

// page.js
Page({
  onLoad() {
    // 1. 监听页面加载
    /* 发送异步请求初始化页面*/
  },
  onShow() {
    // 2. 监听页面显示
  },
  onReady() {
    // 3. 监听初次渲染完成
  },
  onHide() {
    // 4. 监听页面隐藏
    /* opentype = navigate
    ** 保留当前页面，加载新页面（跳转页面）时触发 */
  },
  onUnload() {
    // 5. 监听页面卸载
    /* opentype = redirect/reLaunch/navigateBack
    ** 关闭当前页面时触发 */
  },
  onPullDownRefresh() {
    // 6. 下拉刷新时加载
    /* 在page.json的window参数中配置"enablePullDownRefresh":true
    ** 进行界面刷新操作 */
  },
  onReachBottom() {
    // 7. 监听下拉触底
    /* 需要页面可以上下滚动才能触发，用于加载下一页操作 */
  },
  onShareAppMessage() {
    // 8. 页面滚动时触发
  },
  onResize() {
    // 9.页面尺寸发上改变触发
    /* 横屏/竖屏切换时触发
    ** 需在json中的window字段配置"pageOrientation:"auto"开启 */
  },
  onTabItemTap() {
    // 10. 点击自己的tabBar时触发
  }
});
属性 类型 说明
data Object 页面的初始数据
onLoad Function 监听页面加载
onShow Function 监听页面显示
onReady Function 监听页面初次渲染完成
onHide Function 监听页面隐藏
onUnload Function 监听页面卸载
